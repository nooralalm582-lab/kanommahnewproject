# كن أمة — موقع تعليم القرآن الكريم

موقع عربي (RTL) متجاوب لمنصة "كن أمة"، مبني بـ HTML5 / CSS3 / JavaScript / Bootstrap 5،
ومربوط بقاعدة بيانات حقيقية عبر **Supabase** لتفعيل تسجيل الدخول وإنشاء الحسابات فعلياً.

## محتويات المشروع

```
index.html      الصفحة الرئيسية
login.html      صفحة تسجيل الدخول
signup.html     صفحة إنشاء حساب جديد
style.css       التنسيقات
script.js       منطق الموقع (السلايدر، النماذج، حالة تسجيل الدخول...)
config.js       مفاتيح الاتصال بـ Supabase (تحتاج لتعديلها بنفسك)
```

---

## 1) ربط الموقع بقاعدة بيانات حقيقية (Supabase) — مجاني

الموقع يستخدم **Supabase** (خدمة قاعدة بيانات + تسجيل دخول جاهزة ومجانية) بدلاً من
تشغيل سيرفر خاص بك. هذا يعني أن تسجيل الدخول وإنشاء الحساب سيعملان فعلياً ويخزّنان
بيانات حقيقية، دون أن تحتاج لإدارة سيرفر.

### الخطوات:

1. اذهب إلى **[supabase.com](https://supabase.com)** وأنشئ حساباً مجانياً.
2. أنشئ مشروعاً جديداً (New Project) — اختر اسماً وكلمة مرور لقاعدة البيانات (احتفظ بها).
3. بعد إنشاء المشروع، اذهب إلى: **Project Settings → API**.
4. انسخ القيمتين التاليتين:
   - **Project URL**
   - **anon public** (مفتاح API العام)
5. افتح ملف `config.js` في المشروع، واستبدل القيمتين هنا:

```js
const SUPABASE_URL = "PASTE_YOUR_SUPABASE_PROJECT_URL_HERE";
const SUPABASE_ANON_KEY = "PASTE_YOUR_SUPABASE_ANON_KEY_HERE";
```

> **ملاحظة أمان:** مفتاح `anon` مخصص للاستخدام العلني في المتصفح، ومحمي من جهة
> Supabase عبر نظام الصلاحيات (RLS). يمكنك رفعه إلى GitHub بأمان حتى لو كان
> المستودع عاماً (Public).

### إعداد تأكيد البريد الإلكتروني (اختياري)

افتراضياً، يطلب Supabase من المستخدم تأكيد بريده الإلكتروني عبر رسالة قبل تفعيل
الحساب. إذا أردت تعطيل هذا مؤقتاً للتجربة السريعة:

`Authentication → Providers → Email → عطّل "Confirm email"`

### ما الذي يعمل فعلياً الآن؟

- **إنشاء حساب** (`signup.html`): ينشئ مستخدماً حقيقياً في Supabase مع الاسم ورقم
  الجوال (إن وُجد).
- **تسجيل الدخول** (`login.html`): يتحقق من البيانات الحقيقية المخزّنة.
- **الصفحة الرئيسية**: تتعرف تلقائياً إن كان الزائر مسجّلاً دخوله، وتستبدل زر
  "سجل معنا" بزر "تسجيل الخروج"، وتعرض اسمه في القائمة.

---

## 2) رفع المشروع إلى GitHub

داخل مجلد المشروع، نفّذ الأوامر التالية (مستودع Git مهيأ مسبقاً بداخله):

```bash
# أنشئ مستودعاً جديداً وفارغاً على github.com أولاً، ثم:
git remote add origin https://github.com/USERNAME/REPO-NAME.git
git branch -M main
git push -u origin main
```

استبدل `USERNAME` و`REPO-NAME` ببياناتك.

---

## 3) نشر الموقع مجاناً عبر GitHub Pages

1. من صفحة المستودع على GitHub، اذهب إلى **Settings → Pages**.
2. تحت "Build and deployment" اختر **Deploy from a branch**.
3. اختر الفرع `main` والمجلد `/ (root)`، ثم احفظ.
4. بعد دقيقة تقريباً سيكون موقعك متاحاً على رابط مشابه لـ:
   `https://USERNAME.github.io/REPO-NAME/`

> بديل سريع آخر: يمكنك ربط المستودع بـ **Netlify** أو **Vercel** (تسجيل دخول
> بحساب GitHub مباشرة) للحصول على نشر تلقائي مع كل `git push`، ورابط HTTPS فوري.

---

## 4) التجربة محلياً قبل الرفع

```bash
# من داخل مجلد المشروع
python3 -m http.server 8000
# ثم افتح المتصفح على
http://localhost:8000
```

## ملاحظات

- إن لم تقم بتعديل `config.js` بعد، ستظل النماذج تعمل من ناحية التحقق من الحقول،
  لكنها ستعرض رسالة تنبيه بأن النظام غير مهيأ بعد بدلاً من إنشاء حساب فعلي.
- لإضافة جداول بيانات إضافية مستقبلاً (مثل تتبع تقدّم الطلاب)، أنشئها من
  **Table Editor** في Supabase، وفعّل **Row Level Security (RLS)** عليها دائماً.
